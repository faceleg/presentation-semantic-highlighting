Illegible, but has structure and applies natural language conventions

  • Phrase cueing one operation / action per line
  • Visual-syntactic text formatting "indentation"
  • Phrase cueing: Observed as early in AD 400, when St. Jerome, patron saint of librarians noticed that some scribes had placed phrases on separate lines as a way to help readers read text aloud
  • Visual-syntactic text formatting: more complex than simple phrase cueing because it identifies not only phrases, but also the hierarchical relations across multiple phrases and clauses
    • Eliminates most eyestrain associated with prolonged reading of text
    • Been shown to help university students read challenging material efficiently - code is arguably more challenging?

Syntax Highlighting

Based on fixed keywords for the language:
function var Integer final

Does not help much with the meaning!

  • Much better than nothing
  • Helps identify portions of text important to the language
  • So syntax highlighting helps us keeping an overview or finding the place we’re searching for. However it can _not_ help us actually understanding the code, because by the pure definition of “syntax”, it can only highlight by what the code looks like, not by what the code means, since that requires wider knowledge.

Semantic Highlighting

Semantic Highlighting

Extends Syntax Highlighting

  • Extends Syntax Highlighting
  • Marks individual variables in colour
  • Aids in determining flow of code blocks
  • Alerts programmer of typing errors as they happen (even in dynamic languages)
  • Fills the gap none of us noticed in Syntax Highlighting
